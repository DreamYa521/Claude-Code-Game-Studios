# UX Spec: Deployment Panel（出征面板）

**Screen ID**: `deployment_panel`
**Type**: Modal Overlay
**Input**: Mouse + Keyboard (Esc cancel, Enter confirm)

---

## Purpose

被星图触发的弹出面板。玩家在此决定派出多少兵力、从哪颗星到哪颗星。

---

## Layout

```
┌─────────────────────────────┐
│  出征：地球 → 火星           │  ← 标题
│                             │
│  目标星球：火星 (要塞)        │  ← 目标信息 + 属性
│  敌方驻兵：5                 │  ← 目标驻兵数
│                             │
│  ──────●──────────────────  │  ← 兵力滑块（默认拉到最右=全量）
│        8                     │  ← 当前选中兵力
│                             │
│  出发星剩余：2                │  ← 动态更新：garrison - selected
│                             │
│  [取消]        [确认发兵]     │
│   Esc             Enter      │
│                             │
│  兵力不足或条件不满足          │  ← 红色错误（默认隐藏）
└─────────────────────────────┘
```

---

## Interaction Flow

```
出征面板弹出
    │
    ├── 拖动滑块 → 实时更新兵力数字 + "出发星剩余"
    │
    ├── 点击 [确认发兵] / 按 Enter
    │       │
    │       ├── deploy() 成功 → 面板关闭，命令提交
    │       └── deploy() 失败 → 显示红色错误，面板保持
    │
    └── 点击 [取消] / 按 Esc → 面板关闭，无命令
```

---

## Slider Behavior

| 属性 | 值 |
|------|-----|
| 最小值 | 1 |
| 最大值 | from_planet.garrison（动态） |
| 默认值 | 最大值（全量出兵的常见场景） |
| 步长 | 1 |
| 实时显示 | 选中数字 + 出发星剩余兵力 |

---

## Error States

| 错误 | 显示文字 | 发生条件 |
|------|----------|----------|
| 兵力不足 | "兵力不足或条件不满足" | deploy() 返回 false |
| 非部署阶段 | 面板不弹出（gate 在调用方） | current_phase ≠ DEPLOYMENT |

---

## Keyboard Shortcuts

| 快捷键 | 操作 |
|--------|------|
| Enter | 确认发兵 |
| Esc | 取消出征 |
| ←/→ 方向键 | 滑块 -1/+1（可选，MVP 暂不做） |

---

## Visual Feedback

| 事件 | 响应 |
|------|------|
| 滑块拖动 | 数字即时更新 + "出发星剩余"即时更新 |
| 确认失败 | 红色错误文字出现（面板不关闭） |
| 确认成功 | 面板消失（星图背景自动刷新） |
| 面板打开 | 滑块自动获取焦点（可立即键盘操作） |

---

## Edge Cases

- **出发星只有 1 兵**：滑块 min=1 max=1，锁定为 1（无法拖动）
- **面板打开时游戏状态变更**（如 victory）：面板强制关闭
- **同时打开多个面板**：不允许 — 打开出征面板前先关闭其他弹窗
- **MVP 不做兵种选择**：默认发步兵，UI 不显示兵种下拉
