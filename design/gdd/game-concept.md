## 游戏结构

本游戏由**两个独立模块**组成，分开发、最后合并：

| 模块 | 类型 | 内容 | Demo优先级 |
|------|------|------|------------|
| **Part 1：奴隶→领主** | RPG | 扮演奴隶，反抗推翻领主，故事驱动 | 后做 |
| **Part 2：领主→星际** | 回合制策略 | 代际国王接力，点选发兵征服宇宙 | 先做 |

Part 1 完成后自动过渡到 Part 2，Part 2 沿用 Part 1 的主角血脉和世界状态。

## Part 2 三阶段核心机制

三个阶段共用底层（回合制点选发兵+占点产兵），但每阶段有独立的独特机制：

### 🗡️ 阶段1：领主→国王（人海战术）

**核心机制：养兵博弈**

- 驻守越久的士兵属性越高，但增长曲线递减（前期涨得快，后期几乎不涨）
- 征兵消耗人口，人口影响粮食产出——征太多会崩经济
- 三角克制：步→弓→骑→步
- **博弈**：这批精兵再养一养？还是现在拉出去打？越等收益越低，但不等可能前线扛不住

### ⚙️ 阶段2：国王→统领（武器革命）

**核心机制：双轨制**

- 科技线 OR 人口线，**任意一条达标即可过关**
- 走科技：研发武器打破克制关系（如弓兵配枪反杀骑兵）
- 走人口：继续堆人、养精兵，传统打法依然有效
- 也可两者兼顾
- **自由**：不像阶段1必须精打细算，这里怎么走都行

### 🚀 阶段3：统领→星际（星际战争）

**核心机制：能量的两难**

- 不再征人——星球产**能量**，能量分配到不同用途
- 机器人：稳定消耗战，不耗人口但推进慢
- 量子幽灵/重量级武器：消耗巨大能量，一击改变战局，但风险高
- **博弈**：铺机器兵稳扎稳打？还是憋能量出王炸一锤定音？憋大了可能被消耗死，打小了永远打不完

---

| | 阶段1 | 阶段2 | 阶段3 |
|------|------|------|------|
| **核心资源** | 人口+时间 | 人口/科技（任选） | 能量 |
| **核心机制** | 精兵养成递减曲线 | 双轨达标 | 消耗vs憋大 |
| **博弈点** | 再养一会儿vs现在打 | 走哪条路都能赢 | 铺量vs憋王炸 |

## Elevator Pitch

> 一款**RPG + 策略**的双体量游戏。你从奴隶起步，在 RPG 模式中推翻领主获得自由；然后进入策略模式，一世一代人地征服星球、建设帝国，最终跨越星海统一宇宙。在几代之后，站在先王的墓前，告诉他们星海尽在脚下。

## Core Fantasy

你不是在控制一个国家——你是在**接力一场跨越千年的传承**。

每个国王有生之年能做的事极其有限。你想征服？那科技就得放一放。你想研发？这代可能连疆域都保不住。但没关系——下一代继承你的遗产和遗憾，继续走你没走完的路。

最后，你没有"赢"。你只是站在列祖列宗的墓碑前，告诉他们：**"愚公移山，代代不息——我们到了。"**

---

## Unique Hook

策略游戏 + 国王代际系统。

不是"选科技树"，而是"这一代的国王是个科学家，他死前研发出最后一项关键技术，下一代靠这个突破征服了半个星系"。

每一代国王天赋不同（征服者/科研者/囤积者/外交者），玩家行动消耗国王寿命。你永远做不完所有事——必须选择、放弃、然后交给下一代。

## Player Experience Analysis (MDA Framework)

### Target Aesthetics

| Aesthetic | Priority | How We Deliver It |
| ---- | ---- | ---- |
| **Narrative** | 1 | 代际传承创造独一无二的玩家故事——不是预设剧情，而是你自己的王朝史诗 |
| **Challenge** | 2 | 策略深度：有限寿命内做最优决策，每次选择都有后果 |
| **Fantasy** | 3 | 从奴隶到星际霸主——让玩家真正感到一步步走完了这条史诗路 |
| **Discovery** | 4 | 多层星图：太阳系→不同星系→星系内星球，层层揭开 |
| **Sensation** | Supporting | 像素画风 + 简洁视觉，不抢戏 |
| **Fellowship** | N/A | 单机游戏 |
| **Expression** | Supporting | 不同国王类型、不同发展路径就是玩家的自我表达 |
| **Submission** | Supporting | 回合制，无时间压力，想停就停 |

### Key Dynamics

- 玩家会在地图前沉思"这一代人到底该把时间花在哪"
- 玩家会记住"第三任国王那个天才少年可惜25岁就病死了"
- 玩家之间会讨论"你统一宇宙用了多少代？"
- 每代国王的离世都是一次自然的存档点，玩家放下游戏时会想"下一代该怎么办"

### Core Mechanics

1. **回合制点选发兵**：点己方星球 → 点目标 → 调兵数量 → 确认，回合结束时同时执行
2. **占点产兵系统**：占领星球后自动生产兵力，有上限，平衡进攻与防守
3. **国王代际系统**：每代国王有天赋类型和有限寿命，玩家行动消耗寿命
4. **多层星图**：太阳系（教学）→ 星系选择 → 星系内星球，层层深入
5. **科技代际积累**：科技不是树，是靠国王接力——科研国王去世前踩出关键一步

## Player Motivation Profile

### Primary Psychological Needs Served

| Need | How This Game Satisfies It | Strength |
| ---- | ---- | ---- |
| **Autonomy** | 自由选择进攻目标、国王侧重、科技方向 | Core |
| **Competence** | 策略深度——理解了系统就能打出漂亮的征服路线 | Core |
| **Relatedness** | 对先王和继承人的情感连接——他们不是棋子，是你的血脉 | Supporting |

### Player Type Appeal (Bartle Taxonomy)

- [x] **Achievers** — 统一宇宙是最直白的成就感
- [x] **Explorers** — 多层星图、不同星系的地形和敌人组合，总有新的策略空间可以摸索
- [ ] **Socializers** — 单机无社交（但玩家社群可能自发比较"最快统一用了几代"）
- [ ] **Killers/Competitors** — 纯单机PvE

### Flow State Design

- **上手曲线**：太阳系作为教学关，从几颗星开始，简单直接
- **难度递增**：每个新星系敌人AI更强、地形更复杂、星球分布更难
- **反馈清晰**：每回合看到兵力移动、星球易主、国王年岁增长——所有变化都在地图上
- **失败处理**：输了不可怕——国王自然老死也是一种"完成"，下一代继承继续

## Core Loop

### Moment-to-Moment（30秒）
部署回合：查看星图 → 选择星球 → 决定发兵数量 → 选择目标 → 结束回合 → 看动画执行

### Short-Term（5-15分钟）
一个行星系统的攻略：规划进攻顺序、分配兵力、在每个回合中做"打还是守"的判断

### Session-Level（30-120分钟）
完成一个星系的征服，或者度过一代国王的执政期。国王去世是自然的停止点——你想"下一代该怎么办"但也可以先关游戏

### Long-Term Progression
- **代际层面**：见证你的王朝从奴隶到星际霸主的完整弧线
- **策略层面**：科技代际积累 → 解锁新兵种新能力 → 征服更大星系
- **情感层面**：在先王墓前汇报——你最终能用多少代达成？

### Retention Hooks
- **好奇心**：下一个星系长什么样？有什么敌人？
- **投入感**：这一代的努力不想白费——所有资源都是血的积累
- **支配感**：掌握了系统后，用更少代际完成统一
- **故事**：每个玩家自己的王朝史——没有两局是一样的

## Game Pillars

### Pillar 1: 🕯️ 代际接力

每一代人只能做有限的事——你没时间研发所有科技、攻打所有星球、囤够所有资源。你必须选择，然后交给下一代。

*设计测试*：如果新功能让玩家可以"一代人做完所有"，否决这个功能——它破坏了代际取舍的核心。

### Pillar 2: ♟️ 从容推演

这是下棋，不是劈砍。没有任何东西催你——没有倒计时、没有限时奖励、没有APM要求。

*设计测试*：如果一个机制奖励"快速反应"超过"深思熟虑"，否决。

### Pillar 3: ⚖️ 抉择之重

每个行动消耗国王寿命。出兵？减寿。研发？减寿。囤积？减寿。没有免费的决策。

*设计测试*：如果一个系统允许玩家无成本地反复试错，否决——行动必须有代价。

### Pillar 4: 🌌 自创史诗

没有预设的"主线剧情"。游戏的故事就是你王朝的历史——哪位国王英年早逝、哪一代完成了关键突破、最终站在墓前花了多少代。

*设计测试*：如果一段预设剧情可以替代玩家自发产生的故事，否决——故事必须是玩家自己的。

### Anti-Pillars

- **不做多人**：纯单机，无联网——不分散开发精力
- **不做3D**：2D俯视角像素风，不为画面复杂度牺牲策略深度
- **不做限时**：没有倒计时、没有时间压力——任何催玩家的机制一律不做
- **不做硬核操作**：不需要微操APM，不需要瞬间反应——这是脑力游戏

## Visual Identity Anchor

- **视觉方向**：像素风太空策略——深色宇宙背景 + 彩色星球节点 + 清晰的兵力流动线条
- **视觉规则**：信息高于装饰——玩家一眼看懂兵力分布、星球归属、路线连接
- **支持原则**：
  1. 星球颜色明确区分势力（己方/敌方/中立）
  2. 兵力数字清晰可读，不用图标代替
  3. 动画简洁有限——兵力移动用短线流动，不搞爆炸特效

## Risks and Open Questions

### Design Risks
- 代际系统可能让玩家感到"努力被清零"——新国王继位时的过渡体验需要仔细打磨
- 回合制策略如果没有足够的多样性，中后期可能重复感强

### Technical Risks
- AI敌人策略深度——如果AI太弱，征服感不成立；太强，挫败感强
- 多层星图的UI层级切换需要设计好，避免玩家迷路

### Scope Risks
- 代际系统 + 多层星图 + AI策略 三者叠加，第一个可玩版本需要严格裁剪
- 最大的敌人是完美主义——必须接受Demo阶段很多东西是临时的

### Open Questions
- 国王去世时的过渡演出怎么做？（目前不需要回答，Demo后再说）
- 星系布局是手调还是程序生成？（Demo用手调，以后再说）
- 是否需要事件/随机事件系统？（Demo不做）

## MVP Definition

**核心假设**：玩家能在回合制点选发兵+占点产兵+国王寿命限制的循环中找到策略满足感。

**MVP必做（仅Part 2核心）**：
1. 一张星图，包含行星+卫星，回合制点选发兵
2. 占点自动产兵，有上限
3. 最简单的国王代际系统（一个国王+有限行动次数）
4. 一个AI敌人，会发兵会防守
5. 胜利条件：全歼敌人
6. 纯临时美术资源

**MVP明确不做**：
- Part 1 的 RPG 部分（单独开发，以后合并）
- 完整的代际传承动画和剧情演出
- 多层星图（只有一个星图就够了）
- 外交、科技树、多种国王类型（Demo只有一个国王类型）
- 先王墓地/汇报（Demo不需要）

### Scope Tiers

| Tier | Content | Features | Timeline |
| ---- | ---- | ---- | ---- |
| **MVP** | 1个行星系统（1行星+3卫星） | 核心发兵占点+简易代际 | 1个月 |
| **Vertical Slice** | 太阳系完整 + 1个外星系 | 多层星图 + 多种国王类型 | 2-3个月 |
| **Alpha** | 3-5个星系 | 科技积累 + AI多样化 + 外交雏形 | 6个月 |
| **Full Vision** | 10-15个星系 | 完整代际系统 + 先王汇报 + 全部系统 | 1-2年 |
