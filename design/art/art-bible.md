# Art Bible — 星辰之轭 Part 2

> **Visual Identity Anchor**: 像素风太空策略 — 深色宇宙背景 + 彩色星球节点 + 清晰的兵力流动线条
> **Core Rule**: 信息高于装饰

**Created**: 2026-06-02
**Status**: In Progress

---

## 1. Visual Identity Statement

### One-Line Visual Rule

> **信息高于装饰** — 玩家一眼看懂兵力分布、星球归属、路线连接。任何视觉元素如果不能传递游戏信息，就不该存在。

### Supporting Principles

**原则 1：颜色即归属**
一种颜色代表一个势力。星球颜色是第一眼信息来源 — 不需要读文字、不需要猜。己方/敌方/中立三色足够覆盖率。

- *设计测试*：截一张星图，模糊处理后还能分辨各势力星球 → 通过。

**原则 2：数字不藏**
兵力用数字显示，不用图标代替，不依赖大小缩放。数字位置固定、字体清晰、和背景对比度足够。

- *设计测试*：缩放到 50%，兵力数字依然可读 → 通过。

**原则 3：动画只传达变化**
不做装饰性动画。兵力移动用短线流动表示"部队在转移"。星球占领是一个颜色切换。没有粒子爆炸、没有屏幕震动。

- *设计测试*：第一次看到动画的玩家能准确说出"发生了什么变化" → 通过。

### Pillar Connection

| 原则 | 服务支柱 |
|------|----------|
| 颜色即归属 | 策略深度 — 玩家精力花在决策上，不是花在辨认上 |
| 数字不藏 | 直观战争 — 信息透明 = 决策自信 |
| 动画只传达变化 | 代际史诗 — 安静的氛围让玩家沉浸于"这一代人该做什么"，而不是被特效分心 |

## 2. Color Palette

### Primary Palette

| 色号 | 名称 | 用途 | 含义 |
|------|------|------|------|
| `#0A0E1A` | Deep Space | 星图背景 | 宇宙的深邃和安静 |
| `#4488FF` | Alliance Blue | 己方势力 | 信任、控制、安全 |
| `#FF4444` | Enemy Red | 敌方势力 | 危险、威胁、冲突 |
| `#888888` | Neutral Gray | 中立星球 | 可争夺、未归属 |
| `#FFD700` | Signal Gold | 选中/高亮 | 当前操作目标 |
| `#00CC66` | Confirm Green | 确认/成功 | 操作有效、归属变更完成 |
| `#FF8800` | Alert Orange | 警告/正在交战 | 该星球本回合有战斗 |

### Semantic Color Rules

- 🔵 **蓝** = 你（永远不变）
- 🔴 **红** = 敌人（永远不变）
- ⚪ **灰** = 可抢（永远不变）
- 🟡 **金** = 当前操作（短暂高亮）
- 🟢 **绿** = 刚才发生的正面事件（一闪而过）
- 🟠 **橙** = 需要注意（持续到处理完）

### 色盲安全

- 红/绿冲突：确认操作用 **形状变化**（星球加粗边框）+ 颜色双重提示
- 蓝/紫不敏感：蓝色星球始终带一个 **小徽标/纹理** 区别于任何其他色
- 所有颜色信息都有非颜色备份（数字、位置、边框、纹理）

### UI Palette (与星图区分)

- UI 面板背景：`#1A1F2E`（比星图背景稍亮）
- 按钮常态：`#2A3050` / 悬停：`#3A4070` / 按下：`#4488FF`
- 文字主色：`#E8ECF4` / 次要：`#8899AA`
- 滑块轨道：`#2A3050` / 填充：`#4488FF`

## 3. Shape Language

### 核心几何词汇

**圆形 = 星球**
所有星球都是正圆形。大小区分重要性但不过度（大星球 48px，小卫星 24px）。圆形在深色背景上天生是视觉焦点 — 不需要额外装饰。

**直线 = 连接/路径**
星球间的连线是细直线（1-2px），浅色低透明度。表示"这两颗星球之间可以发兵"。战斗发生时连线高亮闪烁。

**短线流动 = 兵力移动**
部队沿连线移动时，用一排短线沿路径平移。不是整条线变色 — 是短段流动，让人看清楚方向和速度。

**矩形圆角 = UI 面板**
所有 UI 面板用圆角矩形（radius: 4px），和星图的圆形元素区分。UI 是工具，星图是世界 — 形状上就不混淆。

### 层级规则

| 层级 | 形状 | 视觉权重 | 用途 |
|------|------|----------|------|
| 1 | 大圆（48px） | 最重 | 行星，第一眼看到的 |
| 2 | 小圆（24px） | 次重 | 卫星，附属节点 |
| 3 | 细线（1px） | 轻 | 连接线，背景元素 |
| 4 | 矩形面板 | 工具感 | UI 覆盖层 |

## 4. Mood & Atmosphere

### 游戏状态 → 情绪映射

| 状态 | 情绪 | 色调 | 对比度 | 代表元素 |
|------|------|------|--------|----------|
| **DEPLOYMENT（发兵）** | 专注、计算 | 冷蓝主导 | 中高 | 金色选中星球 + 发兵线预览 |
| **EXECUTION（战斗）** | 紧张、期待 | 橙红闪现 | 高 | 交战连线闪烁 + 兵力数字跳动 |
| **CLEANUP（结算）** | 满足/遗憾 | 取决于结果 | 中 | 绿闪占领 / 灰色失守 |
| **TITLE（主菜单）** | 史诗、沉思 | 深蓝+金点缀 | 低 | 星图缓慢旋转 + 标题文字 |
| **VICTORY** | 自豪、释然 | 暖金+深蓝 | 中高 | 金色缓缓扩散覆盖星图 |
| **DEFEAT** | 沉重、但不绝望 | 暗红+黑 | 低 | 红色从边缘向内收缩 |

### 氛围关键词

沉思的、清晰的、安静的、史诗的、克制的

## 5. Character Design Direction

本游戏（Part 2 策略模式）没有可见的角色单位。兵力是抽象的数字，不是小人或模型。

**国王形象**：
- 国王面板显示：代际编号（"第三世"）、名字、寿命条、天赋标签
- 没有国王立绘/头像 — 用文字+图标表示
- 去世时：一个简单的弹窗，文字叙述 + 继位者介绍
- **不在星图上出现任何人物**

**如果有 NPC/敌人国王**：
- 同样抽象化处理 — 名字 + 属性
- AI 敌人没有"脸"，只有势力和行为模式

## 6. Environment Design Language

### 星图就是环境

整个游戏舞台是一张星图。没有行走的地面、没有地形、没有建筑。

**星球节点**：
- 行星：实心圆，半径 24px，颜色 = 势力
- 卫星：空心圆或小实心圆，半径 12px
- 选中状态：金色边框（2px，离圆心 28px）

**连接线**：
- 相邻星球间：细线 1px，颜色 `#334466`（低可见度）
- 该回合有兵力流动的连接：线加粗到 2px，颜色跟随势力
- 可发兵的连接（己方→任意）：鼠标悬停时高亮为金色虚线

**空间感**：
- 无深度/层次 — 纯 2D 俯视
- 星球分布稀疏（3-6颗行星 + 若干卫星足以填满一屏）
- 大片留黑空间 = 宇宙的呼吸感

## 7. UI Visual Language

### 设计原则

**UI 和星图永不混淆**：
- 星图 = 游戏世界（圆形、有机分布、深色背景）
- UI = 玩家工具（矩形、整齐排列、比背景稍亮）

### 字体

- 中文：系统默认黑体（思源黑体/微软雅黑 fallback）
- 数字：等宽字体（Courier New / 内置），保证兵力数字对齐
- 字号层级：标题 24px / 面板文字 14px / 兵力数字 16px / 备注 10px

### 图标

- 扁平化线框图标（16×16px）
- 颜色跟随面板文字色
- 不用实心填充图标（和星球圆形的实心区分开）

### 面板规范

- 背景：`#1A1F2E` + 80%不透明度
- 边框：无，用颜色区分面板边缘
- 内边距：12px
- 按钮：`#2A3050` 常态 / `#4488FF` 按下
- 滑块：暗轨 + 亮填充

### 快捷键标注

所有按钮如果有键盘快捷键，直接标注在按钮文字右侧（灰色小字），例如：
`[结束回合]  Space`

## 8. Asset Standards

### 文件格式

| 类型 | 格式 | 说明 |
|------|------|------|
| 精灵/纹理 | `.png` | 无损压缩 |
| 字体 | `.ttf` / `.otf` | 内置到项目 |
| 音效 | `.ogg` | Godot 推荐 |
| 配置数据 | `.tres` | Godot Resource 格式 |
| 场景 | `.tscn` | Godot 场景文件 |

### 像素规格

- **Tile Size**: 16×16px 基础单位
- **星球半径**: 24px（行星）/ 12px（卫星）
- **UI 面板**: 按像素对齐，不设固定尺寸
- **字体渲染**: 不做像素化处理 — 字体保持清晰锐利

### 命名规范

```
[类别]_[名称]_[变体].[扩展名]
例: planet_earth_allied.png / ui_btn_endturn_hover.png
```

### 性能约束

- 单帧 Draw Calls ≤ 500
- 纹理总内存 ≤ 512MB（2D 像素根本碰不到此上限）
- 所有精灵图集合并（Godot AtlasTexture 或第三方工具）
- 不需要 Mipmap（纯 2D 像素）

### Godot 导入设置

- 纹理 Filter: **Nearest**（保持像素锐利）
- 纹理压缩: **Lossless**（PNG 体积本就很小）
- 字体抗锯齿: **Grayscale**

## 9. Style Prohibitions

### 明确禁止

| ❌ 禁止 | 原因 | 替代方案 |
|---------|------|----------|
| 粒子爆炸特效 | 信息噪音，破坏沉思氛围 | 颜色切换 + 短线流动 |
| 屏幕震动 | 回合制零时间压力 — 震动没有存在的理由 | 颜色高亮眨眼 |
| 3D 透视/深度 | 违反 2D 俯视纯平承诺 | 纯 2D 平面 |
| 渐变/阴影过重 | 像素风要的就是干脆 | 纯色块 + 1px 边框 |
| 圆角 > 8px | UI 失去锐利感和像素风不搭 | 4px 圆角上限 |
| 超过 6 种颜色同屏 | 认知负荷超标，违反"信息高于装饰" | 7色色板内的组合 |
| 兵力用图标代替数字 | 违反"数字不藏"原则 | 数字永远可见 |
| 动画超过 500ms | 回合制，玩家等得起但不该等 | 移动动画 ≤300ms，闪烁 ≤200ms |
| 装饰性 UI 元素 | 每一个像素都要干活 | 去掉它，如果信息不丢就不需要 |

### 一句话总结

> 如果去掉它，玩家还能正确决策 → 它就不该存在。
